## 1. 功能概述 (Feature Overview)

V1.0 版本旨在建立日记应用的核心回顾链路。通过日历矩阵索引，配合轻量化的悬停预览及只读详情页，为用户提供一个结构清晰、视觉整齐的文字记录回顾方案。

## 2. 核心交互流程 (User Journey)

1. **日历视图**：浏览每月的记录分布。
2. **预览弹窗**：鼠标悬停在日期上，快速查阅内容摘要。
3. **新增详情展示页**：点击日期进入，沉浸式阅读当天所有记录（只读）。
4. **更改编辑页入口**：从详情页点击“编辑”按钮进入，进行内容的增删改。

---

## 3. 详细功能说明 (Features)

### A. 日历月视图 (Main Calendar Board)

- **自动排盘**：根据月份动态生成日期矩阵，自动识别当月天数及周起始日。
- **记录标记**：若日期下存在 Block 数据，该格子背景色加深，提供直观的记录密度反馈。
- **月份切换**：支持“上一月/下一月”跳转，实时更新数据索引。

### B. 悬停预览弹窗 (Hover Pre view)

- **即时触发**：悬停于有记录的日期即刻弹出，移开消失。
- **视觉对齐规范**：
  - **左侧标签列**：固定宽度 `60px`，标签文本左对齐。
  - **右侧内容列**：紧接标签后展示文本，形成整齐的垂直视觉引导线。
- **内容逻辑**：
  - **展示顺序**：完全遵循 Block 的创建时间先后顺序（无逻辑干预）。
  - **内容截断**：单条 Block 超过 1 行文本时自动启用 `line-clamp-1` 截断。

### C. 详情展示页 (Detail View - 只读)

- **只读模式**：此页面仅用于查看，不可直接点击编辑文字，防止误操作。
- **页面构成**：
  - **页头**：展示该记录日期的年、月、日及星期。
  - **内容流**：按用户录入的原始顺序依次排列所有 Block，保持与预览弹窗一致的 60px 对齐规范。
- **编辑入口**：页面设有显著的“编辑”按钮，作为进入编辑态的唯一跳转入口。

### D. 编辑页 (Edit Page - 交互)

- **内容管理**：提供完整的 Block 增删改功能。
- **顺序特征**：Block 在数组中的位置决定其展示位置，V1.0 不提供自动排序逻辑。（后续支持按一定顺序排序）
- **数据持久化**：点击“完成”后，将最新的 Block 数组同步至 LocalStorage 并返回详情页。

---

## 4. 业务规则与规范 (Business Rules)

- **数据格式限制**：V1.0 仅识别并存储 `text` 类型数据。
- **视觉栅格系统**：全站（预览/详情/编辑）通用 `60px` 标签对齐规则。
- **空状态逻辑**：点击无记录的日期格子，系统直接引导至该日期的“编辑页”进行初始记录。

## 5. 视觉规范 (Visual Standards)

- **弹窗尺寸**：宽度固定 `320px`，最大高度 `400px`（超出部分可垂直滚动）。
- **色彩定义**：
  - 标签区：`Slate-400`，字重 `Bold`。
  - 内容区：`Slate-600`，字号 `12px`。
