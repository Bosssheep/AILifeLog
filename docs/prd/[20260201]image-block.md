## 1. 项目背景 (Background)

目前的日记应用仅支持纯文本记录。为了丰富记录维度，同时保持日历视角的极致简洁，引入“影像 Block”功能。用户可将多张照片作为独立内容块加入日记，系统自动处理展示逻辑。

## 2. 功能目标 (Goals)

- **原子化**：支持图片作为独立 Block 存在。
- **秩序感**：预览态保持 60px 标签对齐，图片信息高度压缩。
- **高性能**：支持 20+ 张图片，预览态仅显示统计摘要。
- **自动排序**：根据标签权重自动调整日记展示顺序。

## 3. 详细功能说明 (Features)

### A. 数据结构 (Schema)

统一使用 `Block` 对象，通过 `type` 分发逻辑：

- **Text**: `{ "type": "text", "tag": "mood", "content": "..." }`
- **Image**: `{ "type": "image", "tag": "picture", "images": ["url1", "url2", ...] }`

### B. 预览交互 (Calendar Pre view)

- **摘要模式**：识别 `type: image` 后显示：`📷 [标签名]: [N]张照片`。
- **视觉规范**：
  - **标签区**：固定宽度 `60px`，左对齐。
  - **内容区**：显示统计文案，颜色为 `Slate-400`。
- **跳转逻辑**：点击弹窗任意位置跳转至日记详情页。

### C. 编辑管理 (Editor)

- **操作**：支持批量图片上传及内部长按拖拽排序。
- **优化**：前端自动压测图片至 WebP 格式（单图 < 200KB）。
- **封面**：底层预留 `images[0]` 作为封面，日历视图默认关闭展示开关。

### D. 自动排序权重 (Ordering Logic)

展示顺序严格遵循：

1. **Schedule** (计划)
2. **Thoughts** (思考)
3. **Growth** (成长)
4. **Gratitude** (感恩)
5. **Exercise** (运动)
6. **Reading** (阅读)
7. **Picture** (影像) —— _置底展示_

## 4. 非功能性需求

- **滚动支持**：预览弹窗内容过多时自动开启垂直滚动。
- **容错处理**：图片加载失败时不影响摘要数字显示。
